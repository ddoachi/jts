#### Error #9: GitHub Actions Output Format Issues

**Date**: 2025-09-06  
**Error**: 
```
Unable to process file command 'output' successfully.
Error: Invalid format 'brokers-creon'
```

**Root Cause**: **CRITICAL** - `yarn nx show projects --affected --type=lib` returns **multiline output**:
```
brokers-binance
brokers-creon
brokers-kis
...
```

But GitHub Actions `$GITHUB_OUTPUT` expects **single-line values**. When multiline output is passed to `$GITHUB_OUTPUT`, it fails with format errors.

**Investigation**: 
- Analyzed failed CI logs showing multiline project lists
- Tested `npx nx show projects --affected` locally to confirm multiline output
- Confirmed GitHub Actions requirement for single-line output values

**Solution**: Convert multiline output to comma-separated single line:
```bash
# WRONG (causes errors):
AFFECTED_LIBS=$(yarn nx show projects --affected --base=$NX_BASE --head=HEAD --type=lib)

# CORRECT (single-line output):
AFFECTED_LIBS=$(yarn nx show projects --affected --base=$NX_BASE --head=HEAD --type=lib | tr '\n' ',' | sed 's/,$//')
```

**Files Modified**: [`.github/workflows/ci.yml`](.github/workflows/ci.yml)

## Phase 6: Nx Reset Cache Directory Fix (2025-09-06)

### Error #13: Nx Reset Cache Removal Failure

**Date**: 2025-09-06  
**Error**: `Error: ENOTEMPTY: directory not empty, rmdir '/home/runner/work/jts/jts/.nx/cache'`  
**Root Cause**: `nx reset` command fails when trying to remove non-empty `.nx/cache` directories using built-in `rmdir` function  
**Context**: This error occurs in CI environments where cache directories may contain files that can't be removed with simple directory removal commands  

**Investigation**:
- Found `nx reset` usage in two workflow files:
  - [`.github/workflows/setup.yml:129`](.github/workflows/setup.yml) 
  - [`.github/actions/setup-node/action.yml:97`](.github/actions/setup-node/action.yml)
- Error occurs when CI tries to reset Nx workspace but cache directory contains persistent files

**Solution**: Added robust cache cleanup before `nx reset`:
```bash
# BEFORE (fails on non-empty directories):
yarn nx reset

# AFTER (handles non-empty directories):
rm -rf .nx/cache || true
yarn nx reset || true
```

**Files Modified**:
1. [`.github/workflows/setup.yml:129-131`](.github/workflows/setup.yml) - Added `rm -rf .nx/cache || true` before reset
2. [`.github/actions/setup-node/action.yml:97-99`](.github/actions/setup-node/action.yml) - Added same robust cleanup

**Key Improvements**:
- Uses `rm -rf` for forceful cache directory removal
- Added `|| true` to prevent cleanup failures from stopping workflows
- Maintains original `yarn nx reset` functionality while preventing ENOTEMPTY errors
- Ensures CI pipelines can properly clean up Nx cache without failing

**Commit**: `fix(ci): resolve nx reset cache directory removal failures`

### Critical Pattern for Future Nx Cache Management

#### üö® **ALWAYS** Use Robust Cache Cleanup in CI

When using `nx reset` in CI environments:

```bash
# ‚ùå FRAGILE - Fails on non-empty directories
yarn nx reset

# ‚úÖ ROBUST - Handles all cache states
rm -rf .nx/cache || true
yarn nx reset || true
```

This pattern ensures CI reliability regardless of cache directory state and prevents Dependabot PR failures due to cache cleanup issues.
